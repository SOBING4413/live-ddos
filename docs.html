01 Ikhtisar Sistem

Cyber Attack Map merupakan platform intelijen ancaman siber (Cyber Threat Intelligence Platform) yang mengimplementasikan paradigma real-time geospatial threat visualization. Sistem ini dirancang untuk mengagregasi, menormalisasi, dan memvisualisasikan data serangan siber dari multiple authoritative threat intelligence feeds secara simultan.

ℹ️
Tujuan Platform

Menyediakan situational awareness terhadap lanskap ancaman siber global melalui antarmuka kartografis interaktif yang menampilkan attack vectors, threat indicators, dan statistical analytics secara real-time.

Kapabilitas Inti

  • Real-Time Threat Telemetry - Ingesti data streaming dari abuse.ch ecosystem dengan latensi sub-detik
  • Geospatial Attack Mapping - Rendering kartografis interaktif berbasis Leaflet.js dengan custom Canvas overlay
  • Multi-Source Intelligence Aggregation - Orkestrasi pengumpulan data dari heterogeneous threat feeds
  • Statistical Threat Analytics - Dashboard analitik dengan distribusi serangan dan temporal pattern recognition
  • Threat Level Assessment - Algoritma penilaian tingkat ancaman global berbasis composite scoring
  • High-Severity Alert System - Notifikasi real-time untuk insiden severity HIGH dan CRITICAL

02 Arsitektur Teknis

03 Pipeline Data

Pipeline pemrosesan data mengimplementasikan arsitektur multi-stage ETL (Extract, Transform, Load) dengan cascading fallback mechanism untuk memastikan kontinuitas layanan.

Stage 1: Data Acquisition

Sistem menginisiasi koneksi ke threat intelligence APIs melalui CORS-compliant proxy infrastructure. Implementasi menggunakan cascading priority:

  1. Feodo Tracker (Primary) - Botnet C2 server intelligence
  2. URLhaus (Secondary) - Malware URL intelligence
  3. ThreatFox (Tertiary) - IOC database
  4. Simulation Mode (Fallback) - Statistical distribution dari published reports

Setiap endpoint diuji dengan timeout 8-10 detik sebelum failover ke sumber alternatif. Direct fetch dicoba terlebih dahulu, diikuti oleh CORS proxy (allorigins.win, corsproxy.io).

Stage 2: Normalization

Raw data dari heterogeneous sources dinormalisasi ke unified event schema:

Stage 3: Enrichment

Proses enrichment mencakup:

  • Geolocation Resolution - Country code mapping ke koordinat kota menggunakan pre-defined CITY_COORDS database (30+ negara, 50+ kota)
  • Attack Type Classification - Mapping malware family ke attack category (e.g., Emotet = malware, Dridex = malware)
  • Severity Scoring - Penentuan severity berdasarkan threat indicator (online C2 = critical, offline = high)
  • ASN Enrichment - Penambahan Autonomous System Number dan nama dari Feodo Tracker data

Stage 4: Statistical Aggregation

Kalkulasi real-time metrics menggunakan sliding window algorithm:

  • Attacks Per Minute (APM) - Total attacks / elapsed minutes
  • Events Per Second (EPS) - Events dalam 10-detik window / 10
  • Threat Level - Composite score: min(APM * 2, 40) + highSeverityRatio * 60
  • Country Distribution - Running counters per source/target country
  • Route Frequency - Source-to-target country pair counting

04 Threat Intelligence Feeds

Platform mengagregasi data dari ekosistem abuse.ch, sebuah organisasi riset keamanan siber yang diakui secara global dan beroperasi di bawah naungan Bern University of Applied Sciences, Swiss.

Feodo Tracker

⚠️
Primary Data Source

Feodo Tracker merupakan sumber data prioritas utama yang menyediakan IP blocklist dari botnet C2 servers.

Feodo Tracker mengidentifikasi dan melacak infrastruktur Command and Control (C2) dari botnet families termasuk:

  • Emotet - Banking trojan yang berevolusi menjadi malware distribution platform
  • Dridex - Banking trojan dengan kemampuan credential theft
  • TrickBot - Modular trojan untuk reconnaissance dan lateral movement
  • QakBot (Qbot) - Banking trojan dengan worm-like propagation
  • IcedID (BokBot) - Banking trojan yang sering digunakan sebagai initial access broker
  • Pikabot - Loader malware yang muncul sebagai pengganti QakBot

Endpoint: https://feodotracker.abuse.ch/downloads/ipblocklist_recommended.json

Data Fields: ip_address, port, status, hostname, as_number, as_name, country, first_seen, last_online, malware

URLhaus

URLhaus merupakan repository URL yang digunakan untuk distribusi malware, dikurasi oleh komunitas keamanan global. Platform ini mengidentifikasi dan melacak:

  • Malware distribution sites
  • Payload hosting infrastructure
  • Phishing landing pages
  • Exploit kit landing pages

Endpoint: https://urlhaus-api.abuse.ch/v1/urls/recent/limit/20/

Data Fields: id, url, url_status, host, date_added, threat, tags, country

ThreatFox

ThreatFox merupakan platform berbagi Indicators of Compromise (IOC) yang memfasilitasi kolaborasi komunitas keamanan siber. IOC types yang didukung:

  • IP:port - C2 server addresses
  • Domain - Malicious domain names
  • URL - Malware distribution URLs
  • File Hash - MD5/SHA256 hashes of malware samples

Endpoint: https://threatfox.abuse.ch/export/json/recent/

05 Taksonomi Serangan

Sistem mengklasifikasikan ancaman ke dalam enam kategori primer berdasarkan framework MITRE ATT&CK dan ENISA Threat Landscape Report.

DDoS (Distributed Denial of Service)

Definisi: Serangan yang bertujuan membuat layanan tidak tersedia dengan membanjiri target dengan volume traffic yang melebihi kapasitas infrastruktur.

Sub-kategori:

  • Volumetric Attacks - SYN flood, UDP flood, ICMP flood
  • Protocol Attacks - SYN/ACK reflection, Ping of Death
  • Application Layer - HTTP/2 Rapid Reset (CVE-2023-44487), Slowloris, RUDY
  • Amplification - DNS amplification, NTP amplification, SSDP amplification

Common Ports: 80, 443, 53, 8080, 3389

Distribution Weight: 10% (ENISA 2024)

Malware Distribution

Definisi: Proliferasi perangkat lunak berbahaya yang dirancang untuk mengkompromikan confidentiality, integrity, atau availability dari sistem target.

Sub-kategori:

  • Ransomware - LockBit, BlackCat/ALPHV, Cl0p, Royal
  • Banking Trojans - Emotet, Dridex, QakBot, IcedID
  • Infostealers - RedLine, Raccoon, Vidar, Lumma
  • Botnet Infrastructure - C2 servers, loader malware
  • Cryptominers - Unauthorized cryptocurrency mining

Common Ports: 445, 139, 3389, 22, 8443

Distribution Weight: 28% (ENISA 2024)

Phishing & Social Engineering

Definisi: Teknik manipulasi psikologis yang mengeksploitasi human vulnerability untuk mendapatkan akses tidak terotorisasi ke informasi sensitif.

Sub-kategori:

  • Credential Harvesting - Fake login pages, keyloggers
  • Business Email Compromise (BEC) - CEO fraud, invoice fraud
  • Spear Phishing - Targeted attacks pada individu spesifik
  • OAuth Consent Phishing - Abuse of OAuth authorization flows
  • Quishing - QR code-based phishing

Common Ports: 80, 443, 25, 587, 993

Distribution Weight: 22% (ENISA 2024)

Network Intrusion

Definisi: Upaya akses tidak terotorisasi ke sistem atau jaringan melalui eksploitasi authentication mechanisms atau network protocols.

Sub-kategori:

  • Brute Force - Password spraying, credential stuffing
  • Lateral Movement - Pass-the-hash, pass-the-ticket
  • Privilege Escalation - Local/domain privilege escalation
  • APT Campaigns - State-sponsored persistent access

Common Ports: 22, 3389, 5900, 23, 21

Distribution Weight: 18% (ENISA 2024)

Vulnerability Exploitation

Definisi: Eksploitasi kerentanan perangkat lunak yang terdokumentasi (CVE) atau zero-day untuk mendapatkan unauthorized access atau code execution.

Notable CVEs yang Dimonitor:

  • CVE-2024-3400 - Palo Alto PAN-OS command injection (CVSS 10.0)
  • CVE-2024-21887 - Ivanti Connect Secure command injection
  • CVE-2023-46805 - Ivanti Connect Secure authentication bypass
  • CVE-2021-44228 - Log4Shell (Apache Log4j) - residual exploitation
  • CVE-2023-34362 - MOVEit Transfer SQL injection

Common Ports: 80, 443, 8080, 8443, 9200

Distribution Weight: 14% (ENISA 2024)

Reconnaissance & Scanning

Definisi: Aktivitas pengintaian jaringan yang merupakan fase initial dari cyber kill chain, mendahului serangan aktif.

Sub-kategori:

  • Port Scanning - TCP SYN scan, UDP scan, FIN scan
  • Service Enumeration - Banner grabbing, version detection
  • Vulnerability Assessment - Automated vulnerability scanning
  • Network Topology Mapping - Traceroute, DNS enumeration

Common Ports: 22, 80, 443, 3306, 5432

Distribution Weight: 8% (ENISA 2024)

06 Engine Visualisasi

Sistem visualisasi mengimplementasikan dual-layer rendering architecture yang menggabungkan Leaflet.js tile layer dengan custom HTML5 Canvas overlay untuk performa optimal.

Map Configuration

  • Center: [20, 0] (Atlantic-centered global view)
  • Zoom Range: 2 - 8
  • Tile Layer: CartoDB Dark No Labels
  • World Copy Jump: Enabled
  • Max Bounds: [-85, -Infinity] to [85, Infinity]

Attack Arc Rendering

Setiap serangan divisualisasikan sebagai animated arc menggunakan quadratic Bezier curve pada Canvas overlay:

  • Curve Control Point: Perpendicular offset dari midpoint, curvature = min(distance * 0.35, 180px)
  • Travel Animation: easeOutExpo timing function, duration 2200-4000ms
  • Trail Effect: 25-segment gradient trail with progressive opacity
  • Moving Head: White core dot (3-4px) with colored glow (8-12px)
  • Impact Burst: Expanding ring animation at target on arrival
  • Fade Duration: 2000ms with easeInOutCubic

Pulse Markers

Marker pulsasi ditampilkan pada source dan target coordinates:

  • Source Marker: 6px core, 24px pulse ring
  • Target Marker: 8-12px core (severity-scaled), 32-40px pulse ring
  • Animation: pulse-expand keyframe, 2.5s duration, dual-ring with 0.3s stagger
  • Lifetime: 4000ms auto-removal

Performance Optimization

  • Maximum 120 concurrent arcs (oldest removed first)
  • Device Pixel Ratio (DPR) aware canvas scaling
  • requestAnimationFrame-based render loop
  • Automatic arc cleanup when travel + fade duration exceeded
  • Minimum distance threshold (5px) to skip rendering micro-arcs

07 Mode Simulasi

Ketika live API feeds tidak tersedia (umumnya karena browser CORS restrictions), sistem mengaktifkan Simulation Mode yang menggunakan distribusi statistik dari laporan keamanan siber terpublikasi.

⚠️
Transparansi Data

Semua data simulasi dilabeli secara eksplisit dengan prefix [SIM]. Platform tidak pernah memfabrikasi data dan mempresentasikannya sebagai data real. Status "SIMULATION" ditampilkan secara prominent di header dan sidebar.

IP Address Handling

Dalam mode simulasi, sistem menggunakan IP addresses dari RFC 5737 documentation ranges yang secara eksplisit dialokasikan untuk dokumentasi dan contoh:

  • 192.0.2.0/24 (TEST-NET-1)
  • 198.51.100.0/24 (TEST-NET-2)
  • 203.0.113.0/24 (TEST-NET-3)

IP addresses ini tidak pernah di-assign ke real hosts, memastikan tidak ada entitas nyata yang salah diidentifikasi.

Event Generation

Simulation events dihasilkan dengan interval random 800-3300ms menggunakan weighted random selection algorithm. Initial batch sebanyak 15 events dihasilkan saat inisialisasi untuk immediate visual feedback.

08 Glosarium

APT (Advanced Persistent Threat)
Serangan siber yang dilakukan oleh aktor dengan kemampuan tinggi (sering kali state-sponsored) yang mempertahankan akses tidak terotorisasi ke jaringan target dalam jangka waktu yang lama.
C2 (Command and Control)
Infrastruktur server yang digunakan oleh threat actors untuk mengirim perintah ke dan menerima data dari sistem yang telah dikompromikan (bots).
CORS (Cross-Origin Resource Sharing)
Mekanisme keamanan browser yang membatasi HTTP requests dari satu origin ke origin yang berbeda. Menjadi tantangan utama dalam mengakses threat intelligence APIs langsung dari browser.
CVE (Common Vulnerabilities and Exposures)
Sistem identifikasi standar untuk kerentanan keamanan yang dikelola oleh MITRE Corporation. Setiap CVE memiliki ID unik (e.g., CVE-2024-3400).
CVSS (Common Vulnerability Scoring System)
Framework standar untuk menilai severity kerentanan keamanan pada skala 0.0-10.0.
IOC (Indicator of Compromise)
Artefak forensik yang mengindikasikan bahwa sistem telah dikompromikan. Termasuk IP addresses, domain names, file hashes, dan URLs.
MITRE ATT&CK
Knowledge base dan framework yang mendokumentasikan taktik, teknik, dan prosedur (TTPs) yang digunakan oleh threat actors berdasarkan observasi dunia nyata.
Threat Intelligence
Informasi berbasis bukti tentang ancaman siber yang telah dianalisis dan dikontekstualisasi untuk mendukung pengambilan keputusan keamanan.
Zero-Day
Kerentanan keamanan yang belum diketahui oleh vendor perangkat lunak dan belum memiliki patch. Disebut "zero-day" karena vendor memiliki nol hari untuk memperbaiki sebelum eksploitasi.
Kill Chain
Model yang mendeskripsikan tahapan serangan siber: Reconnaissance, Weaponization, Delivery, Exploitation, Installation, C2, Actions on Objectives.

09 FAQ (Frequently Asked Questions)

Apakah data yang ditampilkan merupakan serangan siber nyata?

Tergantung pada mode operasi. Dalam Live Mode, data bersumber dari threat intelligence feeds terverifikasi (abuse.ch) yang menampilkan lokasi real C2 servers dan malware URLs. Dalam Simulation Mode, data dihasilkan berdasarkan distribusi statistik dari laporan keamanan siber terpublikasi dan dilabeli secara eksplisit.

Mengapa platform sering masuk ke Simulation Mode?

Browser memiliki kebijakan CORS (Cross-Origin Resource Sharing) yang membatasi akses langsung ke API eksternal. Threat intelligence APIs dari abuse.ch tidak menyediakan CORS headers, sehingga akses harus melalui proxy. Jika semua proxy tidak tersedia, sistem fallback ke simulation mode.

Bagaimana cara mendapatkan Live Mode secara konsisten?

Untuk akses live data yang konsisten, disarankan untuk mengimplementasikan server-side proxy yang mem-forward requests ke abuse.ch APIs. Ini menghilangkan CORS restrictions karena server-to-server communication tidak tunduk pada same-origin policy.

Apakah IP addresses yang ditampilkan merupakan IP asli?

Dalam Live Mode, IP addresses dari Feodo Tracker merupakan IP asli dari botnet C2 servers. Dalam Simulation Mode, IP addresses diambil dari RFC 5737 documentation ranges (192.0.2.x, 198.51.100.x, 203.0.113.x) yang tidak di-assign ke real hosts.

Seberapa akurat distribusi geografis serangan?

Dalam Live Mode, lokasi source (C2 server) akurat berdasarkan geolocation data. Target distribution menggunakan weighted random berdasarkan published threat reports karena Feodo Tracker hanya menyediakan lokasi C2, bukan target. Dalam Simulation Mode, seluruh distribusi berdasarkan laporan Check Point, Kaspersky, ENISA, dan Fortinet.

Apakah platform ini dapat digunakan untuk keperluan profesional?

Platform ini dirancang untuk keperluan edukasi dan situational awareness. Untuk keperluan profesional threat intelligence, disarankan untuk menggunakan platform komersial seperti Recorded Future, Mandiant Advantage, atau CrowdStrike Falcon Intelligence yang menyediakan data yang lebih komprehensif dan actionable.

10 Implementation Playbook

Bagian ini merangkum langkah implementasi cepat agar tim dapat menjalankan platform secara konsisten dari environment lokal hingga staging.

Quick Start

Bootstrapping Lokal

  1. Jalankan static server pada root project.
  2. Buka index.html untuk landing page dan live.html untuk peta.
  3. Pastikan browser mengizinkan koneksi HTTPS ke endpoint abuse.ch.
Deployment

Hardening Rekomendasi

  • Gunakan reverse proxy internal agar CORS lebih stabil.
  • Tambahkan cache 60-120 detik untuk menghindari rate-limit feed.
  • Aktifkan CSP header dan strict transport security.
Operations

Checklist Monitoring

  • Verifikasi status tiap feed minimal setiap 5 menit.
  • Pantau spike severity HIGH/CRITICAL untuk eskalasi cepat.
  • Review log fallback agar simulation mode tidak dominan.
🧭
Tips Integrasi SOC

Untuk implementasi production, sambungkan event output ke SIEM/SOAR agar notifikasi dan otomatisasi respons bisa diproses dalam satu workflow operasional.
