Metodologi

Arsitektur Pemrosesan Data

Pipeline pemrosesan data multi-stage yang memastikan integritas dan akurasi setiap data point yang divisualisasikan.

01

Data Acquisition

Koneksi ke multiple threat intelligence APIs. Cascading fallback: Feodo Tracker → URLhaus → ThreatFox → Simulation Mode. Timeout 8-10 detik per endpoint.

02

Normalization & Enrichment

Unified event schema. Geolocation resolution, country-to-city mapping, attack type classification, severity scoring, ASN enrichment.

03

Statistical Aggregation

Real-time metrics: attacks-per-minute, events-per-second, country distribution, attack type breakdown. Sliding window algorithm (10s & 60s).

04

Geospatial Visualization

Quadratic Bézier curve pada Canvas overlay. EaseOutExpo timing untuk travel, easeInOutCubic untuk fade. Severity-proportional pulse markers.

05

Simulation Fallback

Distribusi statistik dari Check Point 2024, Kaspersky, ENISA, Fortinet. Data simulasi dilabeli [SIM]. RFC 5737 documentation IPs.

Threat Operations Center

Workflow Deteksi hingga Respons dalam Satu Dashboard

Simulasi alur kerja SOC modern yang menampilkan proses triage, enrichment, validasi IOC, sampai rekomendasi mitigasi dengan konteks yang mudah dipahami tim keamanan.

STEP 01

Signal Ingestion

Event masuk dari feed intelijen eksternal, di-parse secara asynchronous, lalu diverifikasi checksum dan schema compliance.

  • Schema guard & duplicate filter
  • Multi-proxy fallback path
  • Queue prioritization by severity
STEP 02

Context Enrichment

Setiap IOC diperkaya dengan metadata geolokasi, skor ancaman, reputasi ASN, serta korelasi antar attack cluster.

  • Geo + ASN correlation
  • Family classification rules
  • Confidence tagging model
STEP 03

Analyst Decision

Analyst mendapatkan ringkasan yang actionable: host prioritas, indikator critical, dan rekomendasi containment.

  • Severity-first playbook
  • Rule-based escalation
  • Response timeline tracking
